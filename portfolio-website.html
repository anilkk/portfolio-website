Building this portfolio website from scratch

HTML, CSS and JavaScript

As a UX Designer my main responsibility is to conduct user research, understand the user’s challenge to a certain issue, ideate solutions and translate these solutions into concrete wireframes and prototypes. But what happens after that? How are these designs brought to life?

To get a better understanding of what actually happens after the UX and UI Designers job is done - and the work is handed over to a developer in most cases - I decided to take a frontend course to learn and practice HTML, CSS and Javascript. During the course I built my own portfolio website. As a UX Designer, my portfolio is one of the most important platforms for showcasing my skills to potential recruiters, other designers or anyone interested in my work.

Goal

My goal was to enhance my frontend skills to become more confident with cooperating with developers in the future and familiarize myself with industry-standard systems. And of course, build an attractive looking website.

Tools

Atom, Codepen, GitHub, Adobe XD, W3C Validation Service

The challenge

As I started the course I had experience working with HTML but CSS and Javascript were quite new to me. During the course I learnt step by step about containers, writing and pushing commits in GitHub, CSS layouts and styling, mobile-first design, Javascript functions and libraries.

There were definitely a few times during the course that I got frustrated as the code wasn’t behaving like I expected it to but with some patience and the help of my mentor and some very good Youtube videos (which I'm very thankful for) I overcame these challenges and managed to build a website that I’m proud of.

Design & development process

Research & inspiration

I used my current portfolio site on Uxfol.io as an example as I Iiked the template and layout that I have here. I also did a lot of research on other UX Designers portfolios to get some inspiration. I noticed that a common pattern was to briefly present the case studies already on the main page. I liked this approach as then the user can immediately see what the Designer has done without having to click themselves to a separate page.

I wanted the design to be clean, simple yet show some personality. I decided to go for a minimalistic color palette with a bright purple as an interaction color in the buttons and text links.

Information architecture

The important section/pages that I identified for my website and that formed the IA were:

  • Main landing pages
  • Case studies
  • Contact information
  • About me
  • Resume

First wireframes

On the right are few of the first sketches I made of the main page and the about page for desktop and mobile. I wanted the main page to have my name and a short 1-2 sentence introduction about me. After this the case studies would be presented with an image and text. On the desktop these columns are next to each other and on mobile under each other.

Initially I planned that the About page would just have my photo and text. Later I decided to add also a list of my skills and testimonials.

Basic site and page structure with HTML

The structure and IA of the website stayed the same throughout the development process, however, the layout and styling changed several times. I started creating the pages with the help of HTML, making sure that I used division tags often to make it easier later to customize different sections.

I decided to have the contact information in the footer so that it's easily reachable wherever the user is on the site. I linked my CV on the About page.

Defining style and layout with CSS

When I had the HTML pages and elements in place, it was time to incorporate some styling and layout with the help of a CSS stylesheet. I decided to go with a fixed header that would always be visible for the user when scrolling. I had from the very start a mobile-first approach in my design and development process and I used media queries to define how content should be displayed and elements styled at medium and large viewports.

This project helped me understand much better how to do and build a responsive website. Knowing the role of columns and rows in different viewports (and how to define them) was a valuable learning.

Adding JavaScript

For the mobile version I wanted a hamburger menu which I was able to add with JavaScript. I also practiced doing some CSS animations and transitions. I built two simple CSS animations with SVG icons and for the buttons on my site I added transitions for the hover state.

User testing

An important step in any design or development project is user testing. I wanted to test my website with 5 users to identify possible usability issues and get feedback on my portfolio site.

My test objectives were:

  1. Observe how effortlessly users are able to navigate the website - do they immediately know where to go for what information?

  2. Find out any possible usability issues.

  3. See what kind of impression the website and the copy gives of me as a Designer. Does the website showcase my skills and me as a Designer clearly?

I used four scenario tasks and four open ended questions to conduct my user studies.

Improvements after user testing

The usability tests didn't reveal any usability issues. All participants completed the tasks easily and quickly. However, the users had some other valuable feedback which made me do a few improvements such as:

  • I added Testimonials to the About page
  • I rewrote much of the copy to make sure I highlighted the value that I can bring to employers
  • I shortened the "Aida" case study as users found it a bit too long
  • I shortened the intro text on the main page

Code quality, accessibility & cross-browser testing

The last thing I did before the launch of the website was to quality check my code and test it across browsers and devices. I used the linters W3C Validator, linter-stylelint and Standard Javascript style to check the code quality on my HTML, CSS and JavaScript files. I did some minor fixes in my code to remove identified errors or warnings. I also used atom-beautify to help format my code.

The accessibility of my UI elements such as buttons and text links I checked with the color checker and they both passed the AAA compliant.

Lastly I did a quality assurance test in the form of cross-browser testing. I found mainly some minor spacing issues that I fixed.

Final thoughts and learnings

After a steep learning curve in frontend development, I am finally ready and live with my website. I had some challenges on the way (especially with Javascript and I still have much to learn there) but I'm happy with the website I managed to build.

I can say that I have a completely new understanding of the relationship between design and development, and I feel much more confident to talk to developers about my designs after the course. I can also in a whole new way take into consideration the work of the developers when I’m making my designs.

As a UX Designer my aim is not to start writing code myself and building websites but I do find that it is extremely important to be able to speak the same language as developers and understand what restrictions and possibilities there are when it comes to translating wireframes to actual products.

Thank you for reviewing my work!
